GitLab Quick Start Guide

Developer onboarding for the internal GitLab instance

1. Prerequisites

Before you begin, ensure you have:

  • A Google Workspace account on @fedrock.com
  • Tailscale installed (see step 2 — sign in with your @fedrock.com Google account)
  • Git installed on your machine
  • DNS is managed via Cloudflare — no action needed from developers

2. Install & Connect Tailscale

Tailscale provides VPN access to the GitLab instance. There is no other way to reach it — the server has no public endpoint.

Install

PlatformInstall
macOSDownload from Tailscale or brew install --cask tailscale
LinuxInstall script: curl -fsSL https://tailscale.com/install.sh | sh
WindowsDownload from Tailscale

Sign in

  1. Open the Tailscale app
  2. Click "Sign in"
  3. Choose "Sign in with Google"
  4. Use your @fedrock.com Google Workspace account
  5. You'll automatically join the fedrock.com tailnet — no invite code needed

You should see a green "Connected" status in your menu bar / system tray.

Important: You must sign in with your @fedrock.com Google account, not a personal Google account. The tailnet is tied to the fedrock.com domain.

3. Connect to GitLab

  1. Ensure Tailscale is connected (check the icon in your menu bar/system tray)
  2. Open your browser and navigate to: 
    https://code.fedrock.com
  3. Click "Sign in with Google"
  4. Authenticate with your Google Workspace account
  5. You'll be redirected to your GitLab dashboard
First time? Your account is created automatically on first sign-in. You'll have a "Regular" role by default. Ask your admin to grant additional permissions if needed.

4. Set Up Your SSH Key

1Generate an SSH key (if you don't have one)

ssh-keygen -t ed25519 -C "your.name@fedrock.com"

Press Enter to accept the default file location. Set a passphrase when prompted.

2Copy your public key

# macOS
pbcopy < ~/.ssh/id_ed25519.pub

# Linux
xclip -selection clipboard < ~/.ssh/id_ed25519.pub

# Windows (PowerShell)
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard

3Add the key to GitLab

  1. Go to GitLab → User Settings → SSH Keys (click your avatar, top-right)
  2. Paste your public key into the "Key" field
  3. Give it a descriptive Title (e.g., "Work MacBook")
  4. Click "Add key"

4Configure SSH

Add the following to ~/.ssh/config:

Host code.fedrock.com
  HostName code.fedrock.com
  User git
  IdentityFile ~/.ssh/id_ed25519
  AddKeysToAgent yes

5Test the connection

ssh -T git@code.fedrock.com

You should see: Welcome to GitLab, @yourusername!

5. Create a Personal Access Token

Personal access tokens are used for HTTPS Git operations and API access.

  1. Go to User Settings → Access Tokens
  2. Click "Add new token"
  3. Enter a Token name (e.g., "CLI access")
  4. Set an Expiration date (max 365 days)
  5. Select scopes:
    • api — full API access
    • read_repository — read-only Git access
    • write_repository — read-write Git access
  6. Click "Create personal access token"
  7. Copy the token immediately — it won't be shown again
Store your token securely. Use a password manager or your OS keychain. Never commit tokens to version control.

6. Clone Your First Repository

Via SSH (recommended)

git clone git@code.fedrock.com:your-group/your-project.git

Via HTTPS (with personal access token)

git clone https://code.fedrock.com/your-group/your-project.git
# When prompted for password, use your Personal Access Token

To avoid entering your token every time, configure Git credential storage:

# macOS (uses Keychain)
git config --global credential.helper osxkeychain

# Linux (stores in memory for 1 hour)
git config --global credential.helper 'cache --timeout=3600'

7. Troubleshooting

ProblemSolution
Can't reach GitLab URL Check that Tailscale is connected. Try tailscale status in your terminal. If disconnected, run tailscale up.
Google OAuth redirect error Ensure you're signing in with your @fedrock.com Google account, not a personal one. Clear your browser cookies for the GitLab domain and try again.
"Permission denied (publickey)" Verify your SSH key is added to GitLab. Run ssh-add -l to check loaded keys. If empty, run ssh-add ~/.ssh/id_ed25519.
SSH key format not accepted GitLab supports ED25519, RSA (2048+ bits), and ECDSA keys. If using an older RSA key, ensure it's at least 2048 bits: ssh-keygen -l -f ~/.ssh/id_rsa
HTTPS clone asks for username Use your GitLab username (not email). For the password, use your Personal Access Token, not your Google password.
SSL certificate error The GitLab instance uses a valid ACM certificate. If you see SSL errors, your Tailscale connection may have dropped. Reconnect and retry.
"403 Forbidden" on git push Check that your Personal Access Token has the write_repository scope. Check that you have Developer or Maintainer role on the project.

Internal use only. For support, contact your GitLab administrator.
